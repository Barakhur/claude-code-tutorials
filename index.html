Introduction
What is Claude Code?
Understand how Claude Code works and why it's different from other AI coding tools.
⏱ 5 min

Claude Code is Anthropic's terminal-native AI coding assistant. Unlike browser-based chatbots or IDE extensions, it runs directly in your terminal and has full access to your local files.

The core idea

Think of it like having a senior developer sitting next to you in your terminal — one who can read your entire codebase, write and edit files, run commands, and reason through multi-step problems. You describe what you want in plain English, and Claude Code figures out how to do it.

💡 Claude Code launched in February 2025 and hit general availability in May 2025. By late 2025 it had contributed to over $1 billion in annualized revenue for Anthropic — developers are adopting it fast.

How it's different

  • vs GitHub Copilot — Copilot suggests code as you type (~8k token context). Claude Code reasons across your entire project with a 200,000 token context window.
  • vs ChatGPT / Claude.ai — Browser chatbots can't touch your files. Claude Code can read, write, and execute code on your machine.
  • vs Cursor — Cursor is IDE-based. Claude Code lives in the terminal and works anywhere, including SSH sessions.

What it can do

Claude Code is an agentic system — meaning it chains actions together rather than just answering questions. It can:

  • → Read and analyze your entire project structure
  • → Write, edit, and create files across multiple directories
  • → Run terminal commands (tests, builds, linters) on your behalf
  • → Search the web for current best practices
  • → Work through multi-step tasks autonomously while keeping you in control
⚠️ Claude Code always asks permission before modifying files or running commands — unless you explicitly enable "dangerously skip permissions" mode. You're always in control.

Who is it for?

Claude Code is useful for everyone from beginners scaffolding their first project to senior engineers working on large codebases. Non-technical roles like PMs and designers are also using it to build and automate without deep coding knowledge.

02
Setup
Installation & Setup
Get Claude Code installed on Mac, Windows, or Linux in under 5 minutes.
⏱ 8 min

Prerequisites

Before installing, you'll need:

  • 1 A Claude subscription (Pro at $20/month works well for most users) or API access via Anthropic Console.
  • 2 macOS, Linux, or Windows 10+ with PowerShell
  • 3 A terminal (Terminal.app, iTerm2, Windows Terminal, etc.)

Install using the native installer (recommended)

The native installer is the official method and supports auto-updates. Choose your OS:

macOS / Linux # Run this in your terminal:
curl -fsSL https://claude.ai/install.sh | sh
Windows PowerShell irm https://claude.ai/install.ps1 | iex
🍺 You can also install via Homebrew: brew install --cask claude-code or via WinGet: winget install Anthropic.ClaudeCode. Note: these methods require manual updates.

Verify the installation

claude --version
# Should print something like: claude 1.x.x

If the claude command isn't found, restart your terminal. PATH issues after installation are the most common problem — a fresh restart usually fixes it.

Authentication

Run claude in any directory to start the authentication flow:

claude
# Opens a browser window to log in with your Anthropic account

Once authenticated, your credentials are stored. You won't need to log in again unless you switch accounts (use /login to switch).

Billing notes

With a Claude Pro subscription, usage is included in your plan. For API/Console access, charges are per token — the Console auto-creates a "Claude Code" workspace for cost tracking.

⚠️ Some older tutorials still recommend npm install -g @anthropic-ai/claude-code. That method is no longer the recommended approach. Use the native installer above.
03
Basics
Your First Session
Open Claude Code on a real project and make your first code change.
⏱ 8 min

Starting a session

Navigate to any project folder in your terminal and run claude:

cd ~/my-project
claude

You'll see a welcome screen showing your working directory, the model in use, and hints for getting started. The interface is a conversational prompt — type your message and hit Enter.

Step 1 — Explore your codebase

Start by asking Claude to explain your project. It will read your files and build an understanding:

Try these first prompts What does this project do?
Give me an overview of the codebase structure.
Where is the main entry point?

Claude reads your project files on demand — you don't have to manually paste code into the chat.

Step 2 — Make a small edit

Now try a simple change to see the full workflow:

Add a console.log at the start of the main function that prints "App started"

Claude will: analyze the relevant file → show you the proposed change as a diff → ask for your approval → apply it once you say yes.

💡 Key insight: Claude Code always asks permission before modifying files. You review each change before anything is written. You can also enable "Accept all" mode for a session when you're comfortable.

Step 3 — Ask it to fix a bug

The login form throws an error when the email field is empty. Find and fix it.

Talk to Claude like you'd describe the problem to a colleague. You don't need to know which file contains the bug — Claude will find it.

Keyboard shortcuts

  • ↑ Navigate back through previous prompts — even from past sessions
  • Tab Autocomplete file paths and command names
  • Ctrl+C Cancel a running task
  • Esc Back out of the current prompt

Interactive vs one-shot mode

For quick one-off queries, you can skip the interactive session entirely:

-p flag for one-shot mode:

claude -p "What kind of project is this?"
claude -p "List all React components in src/"

One-shot mode is perfect when you want a fast answer without entering an interactive session.

Core Skills
Essential Commands
The slash commands and CLI flags you'll use every single day.
⏱ 8 min

Slash commands (inside a session)

These are typed directly into the Claude Code prompt during an interactive session:

/help
Show all available commands and keyboard shortcuts.
/clear
Reset conversation history to free up context. Use this when starting a new task.
/plan
Enter Plan Mode — Claude outlines steps before doing anything. Great for complex tasks.
/init
Generate a CLAUDE.md file for your project. Run this once when starting with a new codebase.
/compact
Compress conversation history when context is 70-90% full, without losing important details.
/resume
Continue a previous conversation. Useful for picking up long-running tasks.
/login
Switch to a different Anthropic account or re-authenticate.
/exit
Close the current session cleanly.

CLI flags (when launching claude)

Terminal # One-shot query
claude -p "summarize the auth module"

# Continue last conversation
claude -c

# Resume a specific session by ID
claude -r SESSION_ID

# Skip permission prompts (advanced — use with care)
claude --dangerously-skip-permissions
🔥 Pro tip on /clear: Use it every time you start a new task. Old conversation history eats your context window and can confuse Claude. Type /clear, then start fresh. Your CLAUDE.md context is preserved.

Context window management

Claude Code has a 200,000 token context window — far larger than most tools. But it still fills up. Here's the strategy:

  • 0–50% Work freely. No action needed.
  • 50–70% Stay aware. Finish current task soon.
  • 70–90% Run /compact to compress history.
  • 90%+ Run /clear immediately. Responses become erratic above 90%.

Message queuing

You can type multiple prompts and Claude will work through them in order. Instead of waiting, queue your next tasks while the current one runs:

Add unit tests for the user service
# (Claude starts working — you type the next one while it runs)
Also add JSDoc comments to all exported functions
And update the README with the new test commands

Claude is smart about queuing — it won't auto-run the next task if it needs your feedback mid-task.

05
Configuration
CLAUDE.md Setup
Give Claude a persistent memory for your project so it follows your rules every session.
⏱ 8 min

What is CLAUDE.md?

CLAUDE.md is a markdown file at the root of your project. Claude Code automatically reads it when starting a session. Think of it as onboarding documentation for your AI teammate — it tells Claude your conventions, rules, and architectural decisions upfront.

💡 Without CLAUDE.md, Claude makes reasonable guesses about your conventions — sometimes right, often wrong. With it, Claude follows your exact standards from the very first prompt.

Generate one automatically with /init

Start a Claude session in your project, then run:

/init
# Claude analyzes your project and generates a starter CLAUDE.md

Claude will detect your build systems, test frameworks, package manager, and code patterns. Review the output and trim aggressively — bloated CLAUDE.md files cause Claude to ignore your instructions. Keep it concise.

What to include

CLAUDE.md example # Project: My SaaS App

## Stack
- Next.js 14 (App Router), TypeScript, Tailwind
- Supabase for auth + database (Postgres)
- Vitest for unit tests, Playwright for e2e

## Code conventions
- Functional components only, no class components
- Always use TypeScript strict mode
- Named exports, not default exports
- Error handling: always use Result type pattern

## Commands
- Build: npm run build
- Test: npm test (watch: npm run test:watch)
- Lint: npm run lint

## Important
- Never use any as a TypeScript type
- Always write tests for new API routes
- Use server actions for form submissions, not API routes

What NOT to include

  • ✗ Information Claude can already discover by reading your code (file structure, import patterns)
  • ✗ Long explanations — use bullet points only
  • ✗ Secrets or API keys — CLAUDE.md can end up in version control

Multiple CLAUDE.md files

In a monorepo, you can have a root CLAUDE.md plus per-package CLAUDE.md files. Claude reads all relevant ones based on which files it's working with.

⚠️ Most of what /init generates can be inferred from your code already. Review it critically and delete anything Claude would figure out on its own — a lean CLAUDE.md is a powerful CLAUDE.md.

Advanced Basics
Plan Mode & Best Tips
Use Plan Mode for complex tasks and learn the habits of productive Claude Code users.
⏱ 8 min

What is Plan Mode?

Plan Mode is one of Claude Code's most important features. When you enter it with /plan, Claude shifts its behavior: instead of immediately writing code, it analyzes the problem and presents a step-by-step plan for your approval before anything is executed.

/plan
Refactor the authentication module to use JWT tokens instead of session cookies.

Claude will: read relevant files → identify affected areas → outline changes in order → show its reasoning → wait for your approval before executing a single line.

When to use Plan Mode

  • → Complex refactoring — renaming a module imported across dozens of files
  • → Multi-file features — new API endpoint touching routes, controllers, models, and tests
  • → Unfamiliar codebases — understand the impact before making any changes
  • → Learning — Claude's reasoning teaches you how your own codebase fits together

Writing better prompts

The most common mistake is being too vague. Claude infers context well, but it works best when you share your thinking:

❌ Vague prompt Fix the dashboard performance issue.
✅ Better prompt Users are reporting the dashboard takes 15 seconds to load.
Check the API calls on the dashboard page and identify any
performance bottlenecks — look for N+1 queries or missing indexes.

Ask Claude to research things

Claude Code can search the web during a session. Use this explicitly:

I'm not sure about best practices for rate limiting with Redis.
Search the web for current approaches and factor that into the solution.

Top 5 pro tips

  • 1 /clear often. Every new task = fresh context. Don't let old conversation history pollute your prompts.
  • 2 Be specific about tests. Claude loves to mock things in tests. Say "write tests that exercise real functionality without mocking the core logic."
  • 3 Verify outputs. Claude can generate logic errors. Read the diff before approving, especially for complex changes.
  • 4 Use Plan Mode before big changes. It takes 30 seconds and saves hours of debugging bad assumptions.
  • 5 Queue your prompts. While Claude works, type your next task. Claude will queue it and start when the current task completes.
